# 03_GDD 深度编写协议 (Revised: Relaxing & Hardcore Vision)

> [!NOTE]
> 本指令用于指导如何编写一份既具备“编程农场”策略深度，又具备现代独立游戏“愉悦感”的设计文档。

## 1. 技能树与进阶系统 (Technology Tree)
- **模块化解锁**：
  - **初学者阶段**：解锁基础移动、植物识别 API。
  - **进阶阶段**：解锁传感器（检测生长进度）、坐标系统、条件分支。
  - **专家阶段**：解锁并发控制（多无人机调度）、外部数据库集成（作物历史性能分析）。
- **心智负担控制**：每个技能的解锁应配有专门的“微型关卡（Challenge Case）”。

## 2. 创意作物生长逻辑 (Creative Botanical Logic)
- **逻辑即玩法**：
  - **向日葵 (Sunflower)**：不仅是生长，它们按坐标排序收获时有加成（练习排序算法）。
  - **仙人掌 (Cactus)**：相邻格子有作物时会“受伤”（练习空间占用判断）。
  - **蓝莓 (Blueberry)**：需要特定的“采摘节拍”（练习循环与时效判断）。
- **视觉反馈**：作物生长应具有多阶段动态变换，每个阶段提供明确的 UI 提示。

## 3. 引导程序设计 (The Onboarding Program)
- **任务驱动**：不再是枯燥的手册，而是由“主线任务”驱动。
- **交互吸引力**：在代码编辑器侧边栏、地图上方实时浮动“引导弹窗”，指示玩家下一步的尝试。

---

**指令确认：**
在编写具体 GDD 或实现代码时，必须优先考虑：**“这个交互是否让玩家感到愉悦？”** 以及 **“这个逻辑是否在教给玩家某种编程思维？”**
